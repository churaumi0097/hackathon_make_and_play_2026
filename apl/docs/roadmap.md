## ロードマップ / フェーズ分け

一度に全部作らず、コアバリューを最小構成で先に検証する。

- Phase 1（MVP）

プリセット感情入力 + 遠回り強度ダイヤル
感情ベクトル解析（LLM）と象限 → ルートアーキタイプ変換
遠回りルートの生成・描画（安全フィルタ込み）
簡易ナビ + 「最短で帰る」ボタン
到着判定 +「感情に使った時間」表示 + テキストのリザルト共有


- Phase 2

フリーテキスト／音声入力
到着時の感情の再測定（before → after 可視化）
SNSシェア用リザルトカード（画像生成）
“拾いもの”ログ（通過POIの記録・メモ）


- Phase 3

天候連動のルート調整
過去の回り道と感情の変化を振り返る履歴機能
パーソナライズ（好みのPOI傾向の学習）


## 判断基準

Phase を跨いで実装する前に、コアバリュー（無駄な時間を感情のケアに変える）に効くかで優先度を判断する。
